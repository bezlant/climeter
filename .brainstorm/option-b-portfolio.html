I built this because Claude Code
doesn't show your rate limits.

cliMeter is a native macOS menu bar app that tracks Claude Code API usage in real time. Zero dependencies. Pure Swift. Reads credentials from the system Keychain.

The Problem

You don't know you're rate-limited until you're blocked.

Claude Code gives zero indication of how much of your 5-hour session or 7-day weekly limit you've consumed. You find out when your next message gets rejected. cliMeter puts a color-coded progress bar in your menu bar so you always know.

Design Decisions

Why I made these choices

Zero dependencies

No Alamofire, no SwiftLint, no CocoaPods. URLSession handles HTTP, Security.framework handles Keychain, NSWorkspace handles system events. Fewer moving parts = fewer things that break.

Keychain-first, not config-file

Instead of asking users to paste API keys, cliMeter reads the same Keychain entry that Claude Code writes to. The result: run /login once, cliMeter picks it up automatically.

ACL preservation on write-back

When auto-switching accounts, cliMeter writes new credentials to Claude Code's Keychain entry. It uses SecItemUpdate (not delete+add) to preserve the user's "Always Allow" ACL grants.

Screen-lock awareness

macOS locks the login Keychain when the screen locks. Polling during lock = errors. cliMeter listens for DistributedNotificationCenter screen lock/unlock events and pauses accordingly.
